# Resume GitHub Checklist

Use this checklist before adding GitHub to a resume.

## Profile

- [ ] GitHub profile has a professional photo or clear avatar.
- [ ] Profile bio states target role and strongest skills.
- [ ] Location, email, LinkedIn, and portfolio links are current.
- [ ] Profile README exists if the profile repo is available.
- [ ] Pinned repositories show the best 3-6 projects.

## Repository Quality

- [ ] Each pinned repository has a clear `README.md`.
- [ ] README explains the problem, users, features, stack, setup, and screenshots.
- [ ] Project runs locally from documented commands.
- [ ] Dead code, generated files, and secrets are not committed.
- [ ] Commit messages are readable.
- [ ] Repo has a license if you want others to reuse it.

## Resume-Ready Projects

- [ ] At least one full-stack or end-to-end project.
- [ ] At least one project with tests.
- [ ] At least one deployed app, dashboard, API, or published artifact.
- [ ] Projects demonstrate the target role, not only tutorials.
- [ ] Resume bullets include measurable scope or technical depth.

## Minimum Bar Before Sharing

- [ ] No placeholder text remains in pinned project READMEs.
- [ ] Screenshots or demo links work.
- [ ] Install/run instructions were tested from a clean clone.
- [ ] Sensitive files such as `.env` are ignored.
- [ ] The strongest repositories are pinned at the top of the profile.
